# 0x1: 使用内部修正版Engine

# Flutter Engine 替换

由于原生Flutter目前存在诸多问题，公司内部对源码进行了内部修正，目前替换成公司内部库。

# 安装depot_tools

git clone https://https://chromium.googlesource.com/chromium/tools/depot_tools.git
cd depot_tools
pwd  //此处获得的path供稍后使用。简称$path1
vi ~/.bash_profile
//.bash_profile文件进入编辑模式后，下输入以下内容
  #Depot_toolsBegin
  export PATH="$PATH:$path1"
  #DEpot_toolsEnd
//esc后 wq!保存退出
source ~/.bash_profile

# 安装Flutter Engine

使用公司内部engine

# check到内部指定commit

//进入编辑模式后，输入以下
  solutions = [
  {
    "managed": False,
    "name": "src/flutter",
    "url": "https://xxxx/flutterplugin/flutterengine.git/", //内部git
    "custom_deps": {},
    "deps_file": "DEPS",
    "safesync_url": "",
  },
]
//esc, wq!
.gclient sync

# 编译Flutter Engin 工程

# ios 编译

  • 生成engine 工程
//生成ios设备用的未经编译的工程
./flutter/tools/gn --ios --unoptimized

//生成ios设备用的工程，不带符号表
./flutter/tools/gn --ios

//生成release工程
 ./flutter/tools/gn --ios --runtime-mode=release

//生成模拟器版本工程
./flutter/tools/gn --ios --simulator 

//生成模拟器用的未优化版本
./flutter/tools/gn --ios --simulator --unoptimized  

//也可以可以指定cpu
./flutter/tools/gn --runtime-mode=release --ios --ios-cpu=arm64
  • 编译engin工程
//编译relase工程
ninja -C out/ios_release

//编译设备用debug模式
ninja -C out/ios_debug && ninja -C out/host_debug

//编译设备用debug模式,带符号
ninja -C out/ios_debug_unopt && ninja -C out/host_debug_unopt

//编译模拟器用debug模式
 ninja -C out/ios_debug_sim_unopt && ninja -C out/host_debug_unopt
  • 使用编译后的工程产物
//方案1：在工程中使用
flutter run --local-engine-src-path /Users/boo/Documents/engine/src --local-engine=ios_debug_unopt

//方案2：换掉flutter目录里面的engine
 .../flutter/bin/cache/artifacts/engine

# android编译

  • 进入src目录.准备构建文件
./flutter/tools/gn --runtime-mode profile //生成Host编译产物存放文件
./flutter/tools/gn --android --runtime-mode profile //生成Android编译产物存放文件

  • 构建可执行文件

    ninja -C out/host_profile -j 6
ninja -C out/android_profile -j 6
